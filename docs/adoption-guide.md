# Adoption Guide

## Week 1: Pilot

Pick one real cross-repo feature. Use the product-level schema only in a branch. Do not block delivery yet.

## Week 2: PR links

Require each implementation PR to link to:

- parent product change;
- local repo slice;
- implemented requirement/scenario.

## Week 3: CI checks

Add validation for required files/headings. Start warning-only, then make it required.

## Week 4: Retrospective

Review friction:

- Which documents were useful?
- Which sections were noise?
- Did repo-impact catch coordination issues?
- Did agents produce better implementation context?

Tune templates before expanding.
